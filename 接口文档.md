# GoTik 当前接口文档

## 说明

本文档基于当前项目代码自动整理，和目前实际注册的路由保持一致。

当前已开放的接口都属于 `account` 模块，共 3 个：

1. `POST /account/register`
2. `POST /account/login`
3. `POST /account/findByID`

目前接口均为 JSON 请求与 JSON 响应。

## 1. 注册账号

### 接口地址

```text
POST /account/register
```

### 接口说明

创建一个新的账号。

### 请求体

```json
{
  "username": "testuser",
  "password": "123456"
}
```

### 请求字段说明

| 字段 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

### 成功响应

```json
{
  "message": "account created"
}
```

### 失败响应

#### 参数绑定失败

```json
{
  "error": "具体错误信息"
}
```

状态码：`400`

#### 创建账号失败

```json
{
  "error": "具体错误信息"
}
```

状态码：`500`

## 2. 用户登录

### 接口地址

```text
POST /account/login
```

### 接口说明

使用用户名和密码登录，登录成功后返回 token。

### 请求体

```json
{
  "username": "testuser",
  "password": "123456"
}
```

### 请求字段说明

| 字段 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

### 成功响应

```json
{
  "token": "your_jwt_token"
}
```

### 失败响应

#### 参数绑定失败

```json
{
  "error": "具体错误信息"
}
```

状态码：`400`

#### 登录失败

```json
{
  "error": "具体错误信息"
}
```

状态码：`500`

## 3. 按 ID 查询用户

### 接口地址

```text
POST /account/findByID
```

### 接口说明

根据用户 ID 查询账号信息。

### 请求体

```json
{
  "id": 1
}
```

### 请求字段说明

| 字段 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| id | uint | 是 | 用户 ID |

### 成功响应

```json
{
  "id": 1,
  "username": "testuser"
}
```

### 响应字段说明

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| id | uint | 用户 ID |
| username | string | 用户名 |

### 失败响应

#### 参数绑定失败

```json
{
  "error": "具体错误信息"
}
```

状态码：`400`

#### 查询失败

```json
{
  "error": "具体错误信息"
}
```

状态码：`500`

## 测试建议

当前建议按以下顺序测试接口：

1. 先调用 `/account/register` 注册一个新用户
2. 再调用 `/account/login` 获取 token
3. 最后调用 `/account/findByID` 查询用户信息
