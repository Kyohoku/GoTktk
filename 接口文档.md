# GoTik 当前接口文档

## 说明

本文档基于当前项目代码整理，优先覆盖已经完成并用于联调的主要业务接口。

当前已整理的接口包括 `account` 模块、`video` 模块、`like` 模块、`comment` 模块、`social` 模块与 `feed` 模块，共 22 个：

1. `POST /account/register`
2. `POST /account/login`
3. `POST /account/findByID`
4. `POST /account/logout`
5. `POST /video/publish`
6. `POST /video/listByAuthorID`
7. `POST /video/getDetail`
8. `POST /video/uploadVideo`
9. `POST /video/uploadCover`
10. `POST /like/like`
11. `POST /like/unlike`
12. `POST /like/isLiked`
13. `POST /like/listMyLikedVideos`
14. `POST /comment/listAll`
15. `POST /comment/publish`
16. `POST /comment/delete`
17. `POST /social/follow`
18. `POST /social/unfollow`
19. `POST /social/getAllFollowers`
20. `POST /social/getAllVloggers`
21. `POST /feed/listLatest`
22. `POST /feed/listLikesCount`

说明：

1. 除上传接口使用 `multipart/form-data` 外，其余接口当前均为 JSON 请求与 JSON 响应
2. 当前还有一个已跑通但主要用于鉴权验证的接口：`GET /account/me`
3. `feed` 模块当前未挂鉴权中间件，因此可匿名访问；若请求中没有登录态上下文，返回结果中的 `is_liked` 会按匿名用户视角处理

## 1. 注册账号

### 接口地址

```text
POST /account/register
```

### 接口说明

创建一个新的账号。

### 请求体

```json
{
  "username": "testuser",
  "password": "123456"
}
```

### 请求字段说明

| 字段 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

### 成功响应

```json
{
  "message": "account created"
}
```

### 失败响应

#### 参数绑定失败

```json
{
  "error": "具体错误信息"
}
```

状态码：`400`


## 8. 上传视频文件

### 接口地址

```text
POST /video/uploadVideo
```

### 接口说明

上传视频文件并返回可访问的播放地址。该接口需要登录后访问，请在请求头中携带合法 token。

### 请求头

```text
Authorization: Bearer your_jwt_token
Content-Type: multipart/form-data
```

### 表单字段

| 字段 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| file | file | 是 | 仅支持 `.mp4`，最大 `200MB` |

### 成功响应

```json
{
  "url": "http://localhost:8080/static/videos/1/20260312/abc.mp4",
  "play_url": "http://localhost:8080/static/videos/1/20260312/abc.mp4"
}
```

### 失败响应

#### 未登录、缺少文件或文件不合法

```json
{
  "error": "具体错误信息"
}
```

状态码：`400`

#### 保存失败

```json
{
  "error": "具体错误信息"
}
```

状态码：`500`

## 22. 按点赞数倒序查询 Feed

### 接口地址

```text
POST /feed/listLikesCount
```

### 接口说明

按视频点赞数倒序查询视频流；当点赞数相同时，再按视频 ID 倒序稳定分页。该接口当前为公共接口，不需要登录。

### 请求体

第一页示例：

```json
{
  "limit": 10
}
```

下一页示例：

```json
{
  "limit": 10,
  "likes_count_before": 25,
  "id_before": 123
}
```

### 请求字段说明

| 字段 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| limit | int | 否 | 单页条数，当前代码中小于等于 `0` 或大于 `50` 时会自动使用默认值 `10` |
| likes_count_before | int64 | 否 | 点赞数游标，翻页时使用 |
| id_before | uint | 否 | 视频 ID 游标，翻页时与 `likes_count_before` 一起使用 |

### 游标规则

1. `likes_count_before` 和 `id_before` 必须同时传，不能只传其中一个
2. 两者都不传表示请求第一页
3. `likes_count_before` 不能小于 `0`
4. 当 `id_before = 0` 时，只允许 `likes_count_before = 0`

### 成功响应

```json
{
  "video_list": [
    {
      "id": 123,
      "author": {
        "id": 2,
        "username": "author"
      },
      "title": "热门视频",
      "description": "按点赞数排序",
      "play_url": "http://localhost:8080/static/videos/2/20260312/hot.mp4",
      "cover_url": "http://localhost:8080/static/covers/2/20260312/hot.jpg",
      "create_time": 1773277200,
      "likes_count": 25,
      "is_liked": false
    }
  ],
  "next_likes_count_before": 25,
  "next_id_before": 123,
  "has_more": true
}
```

### 响应字段说明

| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| video_list | array | 当前页视频列表 |
| next_likes_count_before | int64 | 下一页点赞数游标；没有更多数据时可能不返回 |
| next_id_before | uint | 下一页视频 ID 游标；没有更多数据时可能不返回 |
| has_more | bool | 是否可能还有下一页 |

### 失败响应

#### 参数绑定失败或游标不合法

```json
{
  "error": "具体错误信息"
}
```

状态码：`400`

#### 查询失败

```json
{
  "error": "具体错误信息"
}
```

状态码：`500`

## 测试建议

当前建议按以下顺序测试接口：

1. 先调用 `/account/register` 注册一个新用户
2. 再调用 `/account/login` 获取 token
3. 调用 `/account/findByID` 查询用户信息
4. 使用登录得到的 token 调用 `/account/logout` 验证退出登录，并重新登录获取新 token
5. 使用登录得到的 token 调用 `/video/uploadVideo` 和 `/video/uploadCover` 获取上传地址
6. 使用上传后的地址调用 `/video/publish` 发布视频
7. 调用 `/video/listByAuthorID` 查询某作者的视频列表
8. 调用 `/video/getDetail` 查询视频详情
9. 使用登录得到的 token 调用 `/like/like` 点赞视频
10. 调用 `/like/isLiked` 检查点赞状态
11. 调用 `/like/listMyLikedVideos` 查看当前用户点赞过的视频
12. 调用 `/like/unlike` 取消点赞后再次检查状态
13. 使用登录得到的 token 调用 `/comment/publish` 发布评论
14. 调用 `/comment/listAll` 查看某视频下的评论列表
15. 调用 `/comment/delete` 删除评论并再次检查列表变化
16. 使用登录得到的 token 调用 `/social/follow` 关注目标用户
17. 调用 `/social/getAllVloggers` 查看自己的关注列表
18. 调用 `/social/getAllFollowers` 查看目标用户粉丝列表
19. 调用 `/social/unfollow` 取消关注后再次检查列表变化
20. 调用 `/feed/listLatest` 验证按发布时间倒序拉取视频
21. 调用 `/feed/listLikesCount` 验证按点赞数倒序拉取视频
