# 3周 Go 短视频后端项目冲刺计划

## 适用背景

- 时间：`2026年3月9日 - 2026年3月31日`
- 当前阶段：中国大陆软件工程大三，目标为暑期实习投递
- 技术方向：Go 后端开发
- 当前基础：Go 基础和常见八股已较熟，`MySQL`、`Redis` 刚开始
- 项目目标：基于 TikTok/短视频 Feed 场景，做一个能写进简历、能经得住面试追问的后端项目

## 总目标

在 `3月31日` 前，至少完成以下 6 个成果：

1. 自己从零写出一个可运行的 Go 短视频后端核心版
2. 跑通 `账号/视频/点赞/评论/关注/Feed`
3. 接入 `MySQL + Redis + RabbitMQ`
4. 至少吃透 3 个亮点：`缓存`、`异步解耦`、`Feed分页`
5. 写出一版简历项目描述
6. 能独立讲清楚“为什么这样设计”

## 核心判断

如果平均能投入 `4-6 小时/天`，这套节奏是来得及的。  
重点不是“完整复刻原项目所有细节”，而是：

1. 先完成后端主链路
2. 再补 Redis 和 RabbitMQ
3. 最后把亮点梳理成自己能讲的内容

## 每天固定学习节奏

每天尽量按以下顺序推进：

1. `30 分钟`：先看设计文档和目标模块，不看源码实现
2. `90-150 分钟`：自己先写
3. `30-45 分钟`：让 Codex review 代码，不要直接代写
4. `30 分钟`：对照原项目实现，看差异
5. `20 分钟`：写笔记，记录今天做了什么、为什么这样做、有哪些坑
6. `10 分钟`：口头复述今天的模块，练表达

## 如果进度落后，怎么砍任务

按以下顺序砍：

1. 先砍前端
2. 再砍文件上传
3. 再砍复杂热榜快照实现，只保留设计理解
4. 最后才砍 MQ

原因：

对暑期实习面试来说，`后端主链路 + Redis + MQ` 的价值远高于页面好看与否。

4. 项目做完了却没有整理面试话术
